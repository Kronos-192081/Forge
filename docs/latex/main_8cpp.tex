Recursively finds and collects all dependencies for a given target.

Determines which nodes in the dependency graph need to be compiled.

This loop iterates over the topologically sorted nodes ({\ttfamily topo\+Sort}) and evaluates whether each node requires compilation based on its dependencies and cache status.


\begin{DoxyItemize}
\item For each node\+:
\begin{DoxyItemize}
\item Retrieves its dependencies.
\item Initializes the {\ttfamily to\+\_\+compile} status for the node as {\ttfamily false}.
\item Iterates over the dependencies\+:
\begin{DoxyItemize}
\item Checks if the dependency is a target ({\ttfamily act\+\_\+targs} or {\ttfamily all\+\_\+targs}).
\begin{DoxyItemize}
\item If the dependency is a target and marked for compilation, marks the current node for compilation and adds it to {\ttfamily compilable\+\_\+nodes}.
\end{DoxyItemize}
\item If the dependency is not a target\+:
\begin{DoxyItemize}
\item Constructs the full file path and checks the cache.
\item If the file is not in the cache, adds it to the cache, marks the current node for compilation, and adds it to {\ttfamily compilable\+\_\+nodes}.
\end{DoxyItemize}
\end{DoxyItemize}
\item If the node is still not marked for compilation\+:
\begin{DoxyItemize}
\item Constructs the full target path and checks if it exists in the filesystem.
\item If the target does not exist, marks the node for compilation and adds it to {\ttfamily compilable\+\_\+nodes}.
\end{DoxyItemize}
\end{DoxyItemize}
\end{DoxyItemize}

\begin{DoxyNote}{Note}
This logic ensures that only nodes with unmet dependencies or missing targets are marked for compilation, optimizing the build process.
\end{DoxyNote}

Parses a file and collects dependencies, updating global tables and variables. 

This function processes a given file, parses its contents, and collects all relevant dependencies, variables, and targets. It ensures that files are processed only once and handles variable redefinitions across files. The function also recursively processes imported files and their dependencies.


\begin{DoxyParams}{Parameters}
{\em filename} & The name of the file to parse and process. \\
\hline
{\em prefix} & A string prefix to prepend to variable and target names (default is an empty string).\\
\hline
\end{DoxyParams}

\begin{DoxyItemize}
\item Updates the {\ttfamily master\+\_\+var\+\_\+tab} with variables from the parsed file.
\item Updates the {\ttfamily master\+\_\+targ\+\_\+tab} with targets from the parsed file.
\item Handles recursive imports and ensures no variable redefinitions occur across files.
\item Processes dependencies for concerned targets and recursively parses imported files.
\end{DoxyItemize}

\begin{DoxyNote}{Note}
If a variable is redefined across files, the function logs an error and exits. 
\end{DoxyNote}

void print\+\_\+banner (\begin{DoxyParamCaption}{}{}\end{DoxyParamCaption})}



Displays a banner with information about the Forge build system. 

This function uses the {\ttfamily tabulate} library to create a styled table that displays a banner with details about Forge, including its purpose, license, and usage instructions. The banner is formatted with various font styles, colors, and alignments to enhance readability.


\begin{DoxyItemize}
\item The banner includes\+:
\begin{DoxyItemize}
\item A title with the name of the build system.
\item A link to the Forge Git\+Hub repository.
\item A brief description of Forge.
\item Highlights such as the required C++ version and license.
\item A quick-\/start instruction.
\end{DoxyItemize}
\item The table is styled using the {\ttfamily tabulate} library with custom colors, font styles, and alignments.
\end{DoxyItemize}

\begin{DoxyNote}{Note}
This function outputs the banner directly to the standard output. 
\end{DoxyNote}
